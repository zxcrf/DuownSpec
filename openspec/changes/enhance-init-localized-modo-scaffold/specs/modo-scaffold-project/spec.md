## ADDED Requirements

### Requirement: MODO Scaffold Asset Set
The system SHALL define a fixed asset set for the MODO scaffold generated by `openspec init --scaffold`.

#### Scenario: Root asset generation
- **WHEN** generating a scaffold project
- **THEN** include root assets required by the MODO workflow, including package metadata, adapter marker files, PRD starter files, and project guidance files
- **AND** align dependency baselines with the MODO scaffold definition used by this fork

#### Scenario: Theme and style generation
- **WHEN** generating a scaffold project
- **THEN** include the required MODO theme files and global styles
- **AND** place them in the scaffold locations expected by the adapter-driven workflow

#### Scenario: Template and business component generation
- **WHEN** generating a scaffold project
- **THEN** include the standard MODO page templates and business components required by the adapter
- **AND** keep them ready for later business implementation without shipping business-specific logic

### Requirement: Empty-Framework Boundary
The system SHALL keep the scaffold limited to framework content and exclude business-specific implementation.

#### Scenario: Excluding business logic from scaffold
- **WHEN** generating a scaffold project
- **THEN** exclude real server actions, production auth/ACL flows, database schemas, migrations, and business-specific pages
- **AND** create empty or placeholder framework directories where those concerns are expected to be implemented later
