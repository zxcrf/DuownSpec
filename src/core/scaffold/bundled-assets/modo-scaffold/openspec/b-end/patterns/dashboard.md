# [Pattern: Dashboard] (统计看板)

## 场景定义
用于高密度的数据监控与业务概览，强调数据的直观对比。

## 1. 布局定义 (Layout Definition)
- **布局名称**：`Dashboard-Layout` (统计看板布局)
- **核心逻辑**：卡片式分块 (`Card-based Layout`)。页面无整体背景色分割，而是通过卡片的白色背景与页面底色（浅灰）形成自然区块。
- **Grid (网格)**：多列复合网格。支持顶部指标区（4 列）与图表统计区（2 列）的灵活组合，确保高密度信息的层级展示。

## 2. 结构框架 (Structure Framework)
- **Metric Grid Area (指标网格区)**：位于页面上半部，采用复合网格高密度排列核心业务指标卡 (`[MetricCard]`)。
- **Analysis Content Area (分析内容区)**：布局的主体部分，包含：
  - **Operation Header (头部操作区)**：通常放置日期筛选 (`DateRangePicker`) 等全局筛选控件，位于区域右上方。
  - **Main Content (主体内容)**：承载图表 (`[ChartPanel]`)、排行 (`[RankList]`) 等深度数据分析模块。
- **Layout Rules (全局布局规则)**：
  - **Gutter (间距)**：卡片间统一使用明显间距，确保区块独立性。
  - **Scroll (滚动)**：内容区整体滚动，内部卡片通常不设独立滚动条。

## 3. 组件清单 (Component List)

### [MetricCard] (指标卡)
- **结构布局 (Composition)**：
  - **Icon Slot**：左侧 48x48px 圆角占位，根据业务类型显示品牌色或语义色图标。
  - **Info Slot**：垂直排布，包含 `Label`（次文本）和 `Value`（特大号数值）。
- **指标变化 (Trend)**：使用语义色标注，上升（成功色）/ 下降（危险色），包含百分比和箭头。
- **交互规范**：鼠标悬停（Hover）时，卡片背景微调或整体微升（lift-up 效果）。

### [ChartPanel] (图表面板)
- **Header (头部)**：
  - **左侧**：主标题（大号字体）+ 辅助信息（可选图标/提示）。
  - **右侧**：操作区，常置按钮组（如：单位切换、全屏、详情跳转）。
- **Body (主体内容)**：
  - **数据展示**：集成 Echarts 或类似图表库，需适配品牌的调色盘。
  - **异常态**：必须包含数据加载中（Skeleton）和空数据（Empty State）的布局。

### [RankList] (排行榜)
- **行构成 (Row Item)**：
  - **索引 (Rank)**：前 3 名高亮（金/银/铜图标），其余名为灰色圆形背景文字。
  - **业务名称**：单行文本溢出。
  - **数据表现**：由 `Value` 和 `Progress Bar` 组成。进度条宽度代表相对于最大值的比例。
- **最大容纳**：单卡片默认展示 Top 5 - Top 7，超出部分建议提供"更多"跳转。

### DateRangePicker (日期筛选器)
- **样式规范**：浅灰色背景胶囊样式（Capsule），无边框或极细边框。
- **快捷预设**：支持"今日"、"近7天"、"近30天"等一键筛选标签。
- **触发逻辑**：点击展开带有日历容器的 Popover。

### MiniTable (简约列表)
- **场景描述**：嵌入在仪表盘卡片内的迷你列表，用于展示更详细的原始记录。
- **结构规范**：
  - **Header**：极简表头，不带背景色，底边框 `1px`。
  - **Density**：高密度展示，行高 `36px`。
  - **Cell**：仅保留核心字段，主文字使用标准正文字号。
- **操作逻辑**：最后一行通常放置"查看全部"跳转链接。

### Role 汇总表

| Role | 说明 | 必选 |
|------|------|------|
| [MetricCard] | 核心业务指标卡 | Yes |
| [ChartPanel] | 图表面板 | Yes |
| [RankList] | 排行榜 | Optional |
| DateRangePicker | 日期筛选器 | Optional |
| MiniTable | 简约列表 | Optional |
