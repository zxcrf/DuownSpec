# [Pattern: Gallery Grid] (卡片目录)

## 场景定义
用于资源的查找、预览与管理，强调"视觉索引"。

## 1. 布局定义 (Layout Definition)
- **布局名称**：`Gallery-Grid-Layout` (画廊网格布局)
- **核心逻辑**：流式网格 (`Fluid Grid`)。内容区被单一维度的卡片矩阵填充，视觉重心在于卡片的"预览图"。
- **Grid (网格)**：
  - **响应式**：当前宽屏状态下展示为 4 列。
  - **比例**：卡片视觉上分为上下两部分，预览区占比约 60%，信息区占比 40%。

## 2. 结构框架 (Structure Framework)
- **[Tabs] (页面标签栏)**：位于页面最顶端，用于多页签切换与内容组织。
- **[Filter] (筛选区)**：提供多种条件组合筛选，底部配有已选条件标签（Tag）展示。
- **Action Bar (业务按钮区)**：位于筛选区下方，左侧放置业务动作按钮，右侧通常预留显示设置入口。
- **Core Card Area (核心卡片区)**：作为页面的主体内容，展示资源的矩阵排列，支持响应式列数调整与内部纵向滚动。

## 3. 组件清单 (Component List)

### [Tabs] (页面标签栏)
- 同 Data-Dense Grid 中的 [Tabs] 规范。

### [Filter] (筛选栏)
- **布局模式**：使用 `layout="inline"`，3 列网格，筛选条件从左向右排列。
- **常用控件组合**：
  - **下拉筛选 ([Select])**：用于"状态"、"部门"、"类型"等维度。
  - **日期范围 (DateRange)**：用于时间维度的过滤。
- **折叠规则**：超过 3 个筛选项时，自动显示"更多筛选"展开按钮。
- **已选条件**：在筛选按钮下方展示当前生效的所有条件 Tag。

### [ActionGroup] (业务按钮组)
- **位置策略**：
  - **主操作**：左对齐（如"+ 新建"）。
  - **视图操作**：(可选) 右对齐（如"刷新"、"设置"）。

### [ResourceCard] (核心卡片组件)
- **容器属性**：
  - **背景色**：白色背景。
  - **滚动**：仅此区域产生纵向滚动条。
- **网格响应式 (Grid System)**：
  - **卡片最小宽度 (Min-Width)**：建议设定为 `280px - 320px`。
  - **列数自动填充 (Auto-Fill)**：使用 CSS Grid: `grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));`
  - **间距 (Gap)**：统一间距。
- **卡片内部结构 (Card Internal Structure)**：
  - **Header (头部预览区)**：
    - **预览图**：固定比例（通常为 16:9），占据卡片上方约 60% 空间。
    - **状态徽标 (Status Badge)**：绝对定位在预览图左上角或右上角。
  - **Body (信息详述区)**：
    - **主标题 (Title)**：加粗文本，大号字体，支持超出显示省略号。
    - **描述文本 (Description)**：次级文本颜色，通常限制显示 1-2 行。
    - **元数据 (Metadata)**：展示创建人、更新时间或计数指标，使用弱化文本样式。
  - **Footer/Action Bar (底部操作栏)**：
    - **操作按钮**：通常采用"更多"图标菜单或 1-2 个常用文字按钮。
    - **Hover 效果**：鼠标悬停在卡片上时，底部操作栏可见性增强或背景微变。

### Role 汇总表

| Role | 说明 | 必选 |
|------|------|------|
| [Tabs] | 页面标签栏 | Optional |
| [Filter] | 页面筛选器 | Yes |
| [Select] | 下拉选择器 | Yes (within Filter) |
